Represents a single state in your FSM.

Template parameters

nimbro_fsm2 heavily uses compile-time verification and introspection. For this reason, you need to specify two template parameters:

  • Derived: This should be your state class.
  • TransitionSpec: Pass in FSM::Transitions with the set of allowed transitions from this state.

Usage

Common usage is to declare states in their own header file:

class Idle;
class Crashed;

class Driving : public FSM::State<Driving, FSM::Transitions<Idle, Crashed>>
{
public:
        void enter() override;
        Transition execute() override;
private:
        std::mt19937 m_randomGenerator;
        int m_count = 0;
};

template<class Derived, class TransitionSpec> template<class ... Args>
void nimbro_fsm2::FSM::State<Derived, TransitionSpec>::display(const std::string& formatString, const Args&... args)

Display values during state execution.

With this method you can visualize high-frequency information from your state execution method. It uses the fmt syntax for format strings. All emitted display strings for one FSM cycle are aggregated and displayed to the user in the nimbro_fsm2 GUI. Additionally, the last collected messages are displayed at state exit, so that one can more easily reason about the conditions that led to the transition. Example:

        std::uniform_real_distribution<float> crashDistribution(0.0, 1.0);
        float crashValue = crashDistribution(m_randomGenerator);
        display("crashValue: {}", crashValue);

template<class Derived, class TransitionSpec> template<class T, class ... Args>
Transition nimbro_fsm2::FSM::State<Derived, TransitionSpec>::transit(Args && ... args)

Perform state transition.

This method is used to perform a state transition. The resulting Transition should be returned from the State::execute method.

Example:

Transition MyState::execute(Driver& driver)
{ return transit<NextState>(); }

Calling transit with a state not mentioned in your TransitionSpec will result in a compilation error.

Any arguments are forwarded to the state constructor.

template<class Derived, class TransitionSpec>
Transition nimbro_fsm2::FSM::State<Derived, TransitionSpec>::stay()

Stay in this state.

This method is used to stay in the current state. The resulting Transition should be returned from the State::execute method.

Example:

Transition MyState::execute(Driver& driver)
{ return stay(); }

template<class Derived, class TransitionSpec>
Transition nimbro_fsm2::FSM::State<Derived, TransitionSpec>::execute() pure virtual

State execution.

You have to implement this function. It is called once per FSM step while this state is active. On each iteration, the method either returns stay to stay in this state, or uses transit to cause a transition to another state:

Transition Driving::execute()
{
        //[display]
        std::uniform_real_distribution<float> crashDistribution(0.0, 1.0);
        float crashValue = crashDistribution(m_randomGenerator);
        display("crashValue: {}", crashValue);
        //[display]

        if(crashValue > 0.8)
        {
                std::cout << "Didn't see that tree coming :-(\n";
                return transit<Crashed>();
        }

        if(++m_count == 10)
        {
                std::cout << "Phew, made it!\n";
                return transit<Idle>();
        }

        // Driving along
        driver().drive(0.0, 1.0);
        return stay();
}

template<class Derived, class TransitionSpec>
DriverClass& nimbro_fsm2::FSM::State<Derived, TransitionSpec>::driver()

Obtain a reference to the global driver object.

You can use this function to access the driver object, which may be used to store information (e.g. discovered objects/poses/etc).

Warning

You cannot use this function in the constructor of your state! The reference is set up after creation of the state object.

template<class Derived, class TransitionSpec>
static auto nimbro_fsm2::FSM::State<Derived, TransitionSpec>::Name constexpr

State name.

This state name is extracted at compile time and stored either as std::string_view (GCC) or a custom static string type (clang). You can use std::string_view{x} to get a runtime string.

        ROSFMT_INFO("Starting in state: {}", std::string_view{my_fsm::Idle::Name});
