nimbro_fsm2::FSM class

Finite State Machine.

This class defines the necessary types and also contains the runtime needed for Finite State Machines. It is templated on a DriverClass type, which can be used by the FSM states to interact with the outside world.

The FSM offers a ROS interface for introspection (information about the current state and state history), as well as control (switching to a new state).

The DriverClass can offer a std::string DriverClass::toString() const method. In that case, the information given by this method is reported in the ROS status message for display in the GUI.

template<class DriverClass> template<class ... StartStates>
void nimbro_fsm2::FSM<DriverClass>::initialize()

Initialize the type system.

This call is used to initialize the compile-time introspection of your states. It should be called with all possible entry states as template parameters.

Example:

        fsm.initialize<my_fsm::Idle>();

Note

All reachable states need to be fully declared (i.e. not only forward-declared) for this to work.

template<class DriverClass> template<class State, class ... Args>
void nimbro_fsm2::FSM<DriverClass>::setState(Args && ... args)

Set FSM state.

This method is used to enter a specific state, continuing execution from there. Any arguments are forwarded to the constructor of the state.

Example:

        fsm.setState<my_fsm::Idle>();

template<class DriverClass>
void nimbro_fsm2::FSM<DriverClass>::step()

Step the FSM.

This will call the State::execute() method on the currently active state and handle the returned Transition object appropriately, either staying in the same or transitioning to another state.

This method will also publish a ROS message on the status topic with the current FSM state and the state history.

template<class DriverClass>
std::string nimbro_fsm2::FSM<DriverClass>::currentStateName() const

Retrieve class name of currently active state.

If the FSM is not running (i.e. no state has been set with setState), this will return std::string{}.

template<class DriverClass>
const Info& nimbro_fsm2::FSM<DriverClass>::stateInfo() const

Information on states & transitions.

This returns a ROS message instance with the list of states and possible transitions.

Note

You will need to call initialize() first, otherwise the result will be empty.
