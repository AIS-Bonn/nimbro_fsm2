nimbro_fsm2

C++17 framework for Finite State Machines with focus on compile-time verification.

This is the documentation of the nimbro_fsm2 package, your friend for writing finite state machines, especially for robotics.

For a hands-on introduction, see the How-To page. Looking for quick API reference? Take a look at the nimbro_fsm2::FSM::State class, which is the heart of the framework.

High-level overview

In robotics, we usually have the property that high-level state is also responsible for control. For example, it is convenient to specify a controller for approaching an object directly in the definition of the respective approach state.

For this reason, nimbro_fsm2 is not event-driven. Rather, it uses a polling design, where the active state gets asked in fixed intervals what the next action is going to be.

Transition Driving::execute()
{
        //[display]
        std::uniform_real_distribution<float> crashDistribution(0.0, 1.0);
        float crashValue = crashDistribution(m_randomGenerator);
        display("crashValue: {}", crashValue);
        //[display]

        if(crashValue > 0.8)
        {
                std::cout << "Didn't see that tree coming :-(\n";
                return transit<Crashed>();
        }

        if(++m_count == 10)
        {
                std::cout << "Phew, made it!\n";
                return transit<Idle>();
        }

        // Driving along
        driver().drive(0.0, 1.0);
        return stay();
}

Ignoring the specifics for a moment, this state defines a behavior that continuously generates driving commands until a number if iterations is reached — or the vehicle crashes. In each of these cases, a state transition to another state is triggered.
